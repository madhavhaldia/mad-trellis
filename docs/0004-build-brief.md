# Part 1 — Global (binds every project)

## Mission & prime directive
BridgeSwarm is the autonomous orchestrator commissioned to BUILD mad-trellis — and mad-trellis is designed to sit *under* orchestrators like BridgeSwarm (GROUNDING Inv. 10). **BridgeSwarm is building its own future floor: a substrate whose invariants would bind BridgeSwarm itself.**

**WHAT:** a single static cgo-free Go binary (CLI + long-lived daemon + a thin **native-Go** cooperative layer — the `mad-trellis mcp` / `mad-trellis hook` subcommands), realizing exactly the 12 projects of 0003 (ids `1,2,3,4,5,6,7,8,9a,9b,10a,10b`) — no more.

**PRIME DIRECTIVE — build TO THE INVARIANTS, not "to make it work."** The 13 invariants are binding constraints. A green test on a feature that quietly weakens an invariant is a **failure**. At every fork, run the decision oracle (GROUNDING L122-130) and obey the asymmetric-error rule (L33-35) as binding text: misclassifying **upward** costs only parallelism (acceptable); **downward** causes corruption (never). **Inv 9 is the soul.** If a feature can't be reconciled with an invariant, the feature loses.

**SCOPE DISCIPLINE:** Layer-1 first and complete; **Layer-2 is NEVER built**, only its seams cut (task-intake kept empty via launcher no-goals/no-dispatch; validation-gate plug-in point in the integrator; scheduler/lease-policy via the classifier's single-trunk-key mapping). "No-brainer" is the phrase that precedes scope creep — when a Layer-2 extension feels irresistible, the invariants bind hardest.

## Trust model & review protocol
**Two trust classes.** A project/component is **REVIEW-GATED** iff it sits on the correctness-critical path — the deterministic lock path (Inv 2), single-arbiter (Inv 5-arbiter), classify-upward (Inv 9), convergent write/trunk + atomic promote/rollback (Inv 6/7), socket-authz/session-identity, or the safety-property conjunction. Else **AUTONOMOUS**.

- **Tests are the contract.** For every review-gated project and every `[GATED]` component, the invariant tests ARE the spec and must be **authored or reviewed by hand — never self-blessed by the implementing agent alone.** Negative/adversarial tests outweigh happy-path, and **each absence-assertion carries a positive control**: inject the forbidden artifact (peer-to-peer channel, MCP-typed daemon method, LLM stub on the lock path, goal-dispatch surface) and assert the check goes **RED**, proving the negative test non-vacuous.
- **Split-trust projects** (4, 5) encode the gate at component granularity via `[GATED]`/`[AUTONOMOUS]` tags on each card: hand-author the gated tests, self-certify the plumbing.
- **One narrow carve-out:** 10b is autonomous EXCEPT the cgo-linkage assertion (a hand-authored guardrail for Inv 3 durability + Inv 2(a) standalone), which may not be weakened by the implementing agent.
- **Joint Inv 2(b)** (no LLM anywhere in the exclusive-access path): cards 1, 2, 3 each hand-author their LOCAL no-inference slice; **conformance-harness (10a) is the named owner of the cross-project conjunction**, into which the three slices roll up.
- **Reviewer authority** = the human maintainer or a Claude reviewer instance — NOT the implementing agent. Submit on the project branch with passing hand-authored invariant tests + an invariant-traceability note (each owned clause → its proving test, incl. negatives) + the chafe log; emit a `submitted-for-review` event and HALT that subtree. Sign-off is an out-of-band signed git tag `reviewed/<id>` — never in a tracked file. **Reviewer signs tests first, then implementation.** No downstream takes a build dependency until sign-off lands; dependents compile against the frozen stub.
- **Autonomous fallback is fail-closed:** if no reviewer is available for a gated project, report BLOCKED — never self-bless a gated test green. A reviewer's invariant objection outranks a passing suite.
- **P0 contract reviewed + frozen before fan-out.**
